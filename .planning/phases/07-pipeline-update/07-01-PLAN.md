Current flow (24 nodes):
  Trigger -> Read Input -> Extract Tab Name -> Get or Create Sheet -> IF Needs Sheet Creation
    -> [true] Create Spreadsheet -> Share Sheet -> Store Sheet ID -> Validate Sheet Access
    -> [false] Pass Existing Sheet -> Validate Sheet Access
  Validate Sheet Access
    -> [success] Merge Validate Data -> Check Tab Exists -> IF Tab Exists
    -> [error] Self-Heal Sheet -> Create Spreadsheet (loop)
  IF Tab Exists
    -> [true] Read Existing IDs -> Filter New Posts -> IF Has New Posts
    -> [false] Create Tab -> Restore Data After Tab Create -> Write Header Row -> Restore Data After Header -> Read Existing IDs
  IF Has New Posts
    -> [true] Build Rows -> Append Rows
    -> [false] Log Skip

Key node signatures (current):

"Filter New Posts" Code node:
  - Reads from $('Read Existing IDs').all() to get existing post IDs
  - Reads from $('Check Tab Exists').first().json for input posts

"Build Rows" Code node:
  - Outputs: Post ID, Author, Post Link, Post Date, Scraped At, _sheetId, _tabName
  - Does NOT currently include column F

"Write Header Row" HTTP Request:
  - URL range: A1:E1
  - Body: ["Post ID", "Author", "Post Link", "Post Date", "Scraped At"]

"Append Rows" HTTP Request:
  - URL range: A:E
  - Body: [Post ID, Author, Post Link, Post Date, Scraped At]

Seen tab schema (from Phase 6 Apps Script):
  Headers: Post ID | Author | Post Link | Post Date | Scraped At | Source Group
  Post ID is column A (used for dedup)

**MODIFY NODE: "Filter New Posts"**

Update jsCode to merge group tab IDs AND Seen tab IDs for dedup (per D-01, D-03, DDP-01):

```javascript
const existingRows = $('Read Existing IDs').all();
const inputData = $('Check Tab Exists').first().json;
const inputPosts = inputData.posts || [];

// Group tab Post IDs (existing behavior)
const existingIds = new Set(existingRows.map(row => String(row.json['Post ID'] || '')));

// Seen tab Post IDs (DDP-01: dedup against Seen tab)
const seenResponse = $('Read Seen Tab IDs').first().json;
if (seenResponse && seenResponse.values) {
  // Skip header row (index 0), add remaining Post IDs
  for (let i = 1; i < seenResponse.values.length; i++) {
    if (seenResponse.values[i][0]) {
      existingIds.add(String(seenResponse.values[i][0]));
    }
  }
}

const newPosts = inputPosts.filter(post => !existingIds.has(String(post.postId)));
if (newPosts.length === 0) {
  return [{ json: { newPosts: [], hasNewPosts: false, sheetId: inputData.sheetId, tabName: inputData.tabName } }];
}
return [{ json: { newPosts, hasNewPosts: true, sheetId: inputData.sheetId, tabName: inputData.tabName } }];
```

When Seen tab doesn't exist, `seenResponse` contains an error object without `values` key — the `if` guard skips it gracefully.
