**CRITICAL DATA FLOW FIX — Node 10a: Merge Validate Data** (Code node — recovers upstream data lost by HTTP response)

The HTTP Request node (Validate Sheet Access) replaces the input payload with the API response body. The response only contains `sheets[].properties.title` — it does NOT contain sheetId, posts, groupUrl, or tabName. This Code node must merge the API response (tab list) with the upstream data from Store Sheet ID / Pass Existing Sheet.

- type: n8n-nodes-base.code, typeVersion: 2
- id: "merge-validate-data-node", name: "Merge Validate Data"
- position: [2400, 200]
- jsCode:
```javascript
// The HTTP response from Validate Sheet Access only has sheets[].properties.title
// Recover sheetId, posts, groupUrl, tabName from the upstream Code nodes
// that fed INTO Validate Sheet Access (Store Sheet ID or Pass Existing Sheet)
const httpResponse = $input.first().json;

// Try Store Sheet ID first (creation path), fall back to Pass Existing Sheet (reuse path)
let upstream;
try {
  upstream = $('Store Sheet ID').first().json;
} catch (e) {
  upstream = $('Pass Existing Sheet').first().json;
}

const sheetId = upstream.sheetId;
const posts = upstream.posts;
const groupUrl = upstream.groupUrl;
const tabName = upstream.tabName;

// Extract tab list from HTTP response
const tabs = (httpResponse.sheets || []).map(s => s.properties.title);

return [{ json: { sheetId, posts, groupUrl, tabName, tabs } }];
```

IMPORTANT implementation notes:
- For IF node branches: index 0 = true branch, index 1 = false branch (follow leads_scrape_apify.json pattern)
- For HTTP Request onError: use `"options": { "response": { "response": { "neverError": true } } }` or set `onError` parameter to `continueErrorOutput` to route 404/403 to error output instead of throwing
- **Self-Heal Sheet -> Create Spreadsheet connection**: This is the self-healing loop per D-04. When the stored sheet ID is inaccessible, Self-Heal Sheet clears static data and returns `{ needsCreation: true, posts, groupUrl, tabName }`. The output connects to Create Spreadsheet, which creates a new sheet. The flow then continues through Share Sheet -> Store Sheet ID -> Validate Sheet Access as normal. Note: Create Spreadsheet ignores the input JSON fields (it just creates a new sheet named "Facebook Leads"), so the posts/groupUrl/tabName in Self-Heal output are carried forward via `$('IF Needs Sheet Creation').first().json` references in Store Sheet ID. **However**, when entering from Self-Heal, `IF Needs Sheet Creation` may not have run in this execution path. To handle this, Store Sheet ID should fall back: use `$('Self-Heal Sheet').first().json` as a secondary source. Update Store Sheet ID jsCode to:
```javascript
const staticData = $getWorkflowStaticData('global');
const newSheetId = $('Create Spreadsheet').first().json.spreadsheetId;
staticData.sheetId = newSheetId;
// Try IF Needs Sheet Creation (normal path) then Self-Heal Sheet (recovery path)
let prev;
try {
  prev = $('IF Needs Sheet Creation').first().json;
} catch (e) {
  prev = $('Self-Heal Sheet').first().json;
}
return [{ json: { sheetId: newSheetId, posts: prev.posts, groupUrl: prev.groupUrl, tabName: prev.tabName } }];
```
- The `_sheetId` and `_tabName` fields in Build Rows output will be auto-mapped by Google Sheets node — but they won't match header columns, so they'll be ignored. Alternatively, exclude them and reference via expression. Safer approach: use `columns.mappingMode: "defineBelow"` with explicit column mapping in Append Rows instead of autoMap, to avoid writing _sheetId/_tabName as columns. Use:
  - column A: `={{ $json['Post ID'] }}`
  - column B: `={{ $json['Author'] }}`
  - column C: `={{ $json['Post Link'] }}`
  - column D: `={{ $json['Post Date'] }}`
  - column E: `={{ $json['Scraped At'] }}`
  And pass sheetId/tabName via expression referencing the Filter New Posts node: `={{ $('Filter New Posts').first().json.sheetId }}`
