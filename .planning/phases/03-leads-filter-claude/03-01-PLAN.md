The workflow has 6 nodes in this chain:

**Node 1: "Execute Workflow Trigger"**

**Node 2: "SplitInBatches"**
- Type: `n8n-nodes-base.splitInBatches`
- typeVersion: 3
- Parameters: `batchSize: 10`
- Option: `reset: false`
- Position: [500, 300]

**Node 3: "Build Batch Message"**
- Type: `n8n-nodes-base.code`
- typeVersion: 2
- Position: [750, 300]
- jsCode (exact):
```javascript
const items = $input.all();
const posts = items.map(item => ({
  id: item.json.postId,
  text: item.json.postText,
  author: item.json.authorName
}));
return [{
  json: {
    batchMessage: JSON.stringify(posts),
    originalItems: items.map(i => i.json)
  }
}];
```

**Node 4: "Claude API Request"**
- Type: `n8n-nodes-base.httpRequest`
- typeVersion: 4
- Position: [1000, 300]
- method: POST
- url: `https://api.anthropic.com/v1/messages`
- authentication: `genericCredentialType`
- genericAuthType: `httpHeaderAuth`
- sendHeaders: true
- headerParameters:
  - name: `anthropic-version`, value: `2023-06-01`
  - name: `Content-Type`, value: `application/json`
- sendBody: true
- contentType: `json`
- body (as expression — the system prompt must be included VERBATIM, not referenced):
```
={
  "model": "claude-3-5-sonnet-20241022",
  "max_tokens": 512,
  "system": "You are a lead qualification expert for a Family Law Attorney in Israel.\nYou read Hebrew fluently and understand conversational, colloquial, and formal Hebrew.\n\nA LEAD is a Facebook post where a real person is actively seeking help related to:\n- Divorce or separation (גירושין, פרידה)\n- Child custody disputes (משמורת ילדים)\n- Alimony or financial disputes in marriage (מזונות, הסכם ממון)\n- Restraining orders or domestic violence (צו הגנה, אלימות במשפחה)\n- Family legal advice requests (ייעוץ משפטי משפחתי)\n- Property disputes in marriage or divorce (חלוקת רכוש)\n\nNOT a lead:\n- Posts offering legal services (attorneys advertising)\n- General news, opinion, or informational posts\n- Posts that mention these topics without expressing a personal need\n- Spam, promotions, or unrelated content\n\nRespond ONLY with valid JSON in this exact format:\n{\"relevant_post_ids\": [\"id1\", \"id2\"]}\n\nIf no posts are relevant, return: {\"relevant_post_ids\": []}",
  "messages": [{"role": "user", "content": "{{ $json.batchMessage }}"}]
}
```
- options: timeout 60000
- credentials block: `httpHeaderAuth: { id: "anthropic-header-auth-credential", name: "Anthropic API" }`
- (per FLTR-03)

**Node 5: "Parse and Filter Response"**
- Type: `n8n-nodes-base.code`
- typeVersion: 2
- Position: [1250, 300]
- jsCode (exact — includes try/catch for FLTR-06, empty handling for FLTR-07):
```javascript
const responseText = $input.first().json.content[0].text;
const originalItems = $('Build Batch Message').first().json.originalItems;

let parsed;
try {
  parsed = JSON.parse(responseText);
} catch (e) {
  throw new Error('Claude returned malformed JSON: ' + responseText.substring(0, 200));
}

const relevantIds = parsed.relevant_post_ids;
if (!Array.isArray(relevantIds)) {
  throw new Error('Claude response missing relevant_post_ids array: ' + responseText.substring(0, 200));
}

if (relevantIds.length === 0) {
  return [];
}

const idSet = new Set(relevantIds);
const filtered = originalItems.filter(item => idSet.has(item.postId));
return filtered.map(item => ({ json: item }));
```
- (per FLTR-04, FLTR-05, FLTR-06, FLTR-07)

**Node 6: "No Operation" (output collector)**
- Type: `n8n-nodes-base.noOp`
- typeVersion: 1
- Position: [500, 500]
- This node receives the SplitInBatches "done" output (output index 0) to collect all aggregated results

**Connections:**

1. Execute Workflow Trigger -> SplitInBatches (main[0])
2. SplitInBatches output 1 (batch) -> Build Batch Message (main[0])
3. Build Batch Message -> Claude API Request (main[0])
4. Claude API Request -> Parse and Filter Response (main[0])
5. Parse and Filter Response -> SplitInBatches (main[0]) — LOOP BACK
6. SplitInBatches output 0 (done) -> No Operation (main[0])

Note on SplitInBatches connections: output index 0 = "done" (all batches processed), output index 1 = "each batch". The loop-back from Parse and Filter Response goes to input of SplitInBatches.

Settings: `{ "executionOrder": "v1" }`, active: false, meta: `{ "instanceId": "" }` — matching Phase 2 pattern exactly.
